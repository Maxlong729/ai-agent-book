## 二、贯穿人物

**小陈** —— 一个做了半年生活类短视频的博主。

- 粉丝量：3万（不算小白，但也不是大V）
- 痛点：选题靠灵感、脚本靠手写、剪辑靠熬夜。断更三天就掉粉，但连更三天就崩溃。
- 转折：他在一次偶然中尝试用流程化方式管理内容生产，发现日更只需不到2小时，断更率降为零。
- 作用：小陈不是一个"完美案例"，而是一个"真实演进过程"的载体。读者跟着他一起踩坑、一起优化。

> **设计意图**：小陈贯穿全章，避免案例篇变成"干巴巴的流程罗列"。用他的视角带出每一个知识点，让读者有代入感。

### 7.1 将一条视频拆成十个固定步骤（约 2000 字）

**核心论点**：做视频不是一个"创作行为"，而是一个"生产流程"。把它拆成固定步骤后，你会发现其中 70% 的环节根本不需要灵感。

**叙事线**：小陈把自己做一条视频的全过程录屏回看，发现他在10件事之间反复跳跃。于是他尝试按时间线把这些事排成一条直线。

#### 7.1.1 前期四步：选题 → 定角度 → 起标题 → 写脚本

| 步骤 | 做什么 | 常见坑 | 原子级拆解示范 |
| :--- | :--- | :--- | :--- |
| **选题** | 确定今天拍什么话题 | 刷1小时热搜，什么都想拍，什么都没定 | ① 打开3个热点源（抖音热搜/微博热搜/同领域对标账号）→ ② 花10分钟浏览 → ③ 列出3个备选 → ④ 用"我的粉丝会不会划走？"标准选1个 |
| **定角度** | 确定切入角度和核心观点 | 话题太大，说不出重点 | ① 对选定话题问自己"我和别人的区别是什么？" → ② 用一句话写出核心观点 |
| **起标题** | 写视频标题和开头钩子 | 标题不吸引人，完播率低 | ① 写5个备选标题 → ② 用"3秒测试"筛选（如果这是你刷到的标题，你会点吗？） → ③ 选定1个 |
| **写脚本** | 写出完整的口播/旁白脚本 | 想到哪说到哪，逻辑混乱 | ① 用"痛点→方案→行动"三段式搭骨架 → ② 每段限100字 → ③ 通读一遍确认口语化 |

**回扣方法论**：这一步就是第3章的「Step 1：拆解动作」和第4章的「原子级拆解」。小陈用的是同样的方法——把"做视频"这个大任务拆成动词短语。

#### 7.1.2 中期三步：找素材/生成素材 → 设计分镜 → 准备配音

| 步骤 | 做什么 | 原子级拆解示范 |
| :--- | :--- | :--- |
| **找/生成素材** | 准备视频需要的画面素材 | ① 根据脚本逐段标注需要的画面类型 → ② 实拍素材：按清单拍摄 → ③ 网络素材：从3个免费素材站搜索 → ④ AI生成素材：用图像工具按提示词生成 |
| **设计分镜** | 把脚本变成"画面脚本" | ① 把脚本按句子切割 → ② 每句话标注对应画面（实拍/素材/文字特效） → ③ 形成一张「分镜表」 |
| **准备配音** | 录制或生成旁白 | ① 自己录：按脚本朗读，每段录2遍取最好的 → ② AI配音：把脚本喂给TTS工具，选合适的音色 |

#### 7.1.3 后期三步：剪辑 → 制作封面 → 发布并记录数据

| 步骤 | 做什么 | 原子级拆解示范 |
| :--- | :--- | :--- |
| **剪辑** | 把素材和配音拼成成品 | ① 按分镜表把素材拖入时间线 → ② 对齐配音 → ③ 加字幕 → ④ 加背景音乐 → ⑤ 导出 |
| **制作封面** | 做一张吸引点击的封面图 | ① 从视频中截取最有冲击力的一帧 → ② 加标题文字（大字+对比色） → ③ 确保手机端清晰可辨 |
| **发布+记录** | 发布并跟踪数据 | ① 写发布文案（含话题标签） → ② 选择发布时间段 → ③ 发布后24小时记录播放量、完播率、互动率 → ④ 填入数据表 |

**小结**：展示完整的10步流程图（一张图），并用第3章的 I/O 表格格式标注每一步的输入和输出，做一次「接口对齐检查」。

**【心法】**：你以为"做视频"是一件事，其实它是十件事。把它拆开之后你会发现：只有选题和定角度需要灵感，剩下的八件事都是流程。

### 7.2 哪些环节最适合交给 Agent（约 2000 字）

**核心论点**：不是所有步骤都适合交给 AI。你需要像第4章教的那样，做一次「人机分工」，把合适的环节交给合适的执行者。

**叙事线**：小陈拿着上面的十步流程图，逐一标注：这一步该人做还是 Agent 做？他发现，10步里有6步可以全部或部分交给 Agent。

#### 总览分工表（全章核心图表之一）

| 步骤 | 执行者 | Agent 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| ① 选题 | Agent + 人审核 | 信息类 Agent | Agent 扫热点和竞品，人最终拍板 |
| ② 定角度 | **人** | — | 核心创意决策，不可外包 |
| ③ 起标题 | Agent + 人筛选 | 写作类 Agent | Agent 给5个备选，人选1个 |
| ④ 写脚本 | Agent + 人修改 | 写作类 Agent | Agent 出初稿，人改口语化 |
| ⑤ 找/生成素材 | Agent + 人 | 图像类 Agent | AI生成部分素材，实拍部分人来 |
| ⑥ 设计分镜 | **人** | — | 视觉创意判断，暂不交AI |
| ⑦ 准备配音 | Agent | TTS工具 | 纯执行，交给AI |
| ⑧ 剪辑 | **人** | — | 剪辑节奏感需要人把控 |
| ⑨ 制作封面 | Agent + 人筛选 | 图像类 Agent | Agent 出3版，人选1版 |
| ⑩ 发布+记录 | Agent + 自动化脚本 | — | 数据记录交给脚本自动化 |

**回扣方法论**：这张表就是第4章「第四步：为每一步分配最合适的执行者」的实战版。提醒读者翻回第4章的「任务分配矩阵」（创意决策类/海量处理类/标准写作类/精准计算类）。

#### 7.2.1 热点扫描与竞品拆解 → 信息类 Agent

- 展示一份完整的「选题 Agent JD」（基于第5章的万能模板填写）
- 重点：限定信息来源（如只看抖音/小红书/B站的特定类目）、限定输出格式（每条包含话题+热度+对标视频链接+内容摘要）
- 一个简短的 Prompt 示范 + Agent 输出示范

#### 7.2.2 脚本初稿与标题备选 → 写作类 Agent

- 展示「脚本 Agent JD」
- 重点：输入投喂必须包含（选题+角度+目标人群+风格参考视频链接），否则AI会写出"正确但无聊"的脚本
- 强调第5章的「写作类 Agent 死规矩：结构第一，文采第二」——先让AI把骨架填满，人再改文采

#### 7.2.3 分镜草图与封面设计 → 图像类 Agent

- 简要介绍图像类 Agent 的使用场景（不深入工具操作，点到为止）
- 重点：封面设计的 Prompt 结构（风格词+色调+文字位置+情绪关键词）
- 一个简短的输入/输出示范

**【心法】**：Agent 不是来取代你的创作力的，而是来承接你创作力之后的「执行苦力」。你负责决定"说什么"，Agent 负责"怎么说"和"怎么做"。

### 7.3 日更账号的标准工作流长什么样（约 2500 字）

**核心论点**：日更不是拼体力，是拼流程。有了标准化工作流后，日更的难度和一周三更没有本质区别。

**叙事线**：小陈把上面的十步分工方案，按时间轴排成一天的工作节奏。他发现，如果流程设计得当，他只需要每天投入不到2小时就能完成日更。

#### 7.3.1 早上（9:00-10:00）：热点监测与选题流程

**工作流描述**：
1. [Agent] 选题 Agent 在早上8点自动运行，扫描昨晚到今早的热点，输出「今日选题推荐清单」（3-5条）
2. [人] 9:00 小陈打开清单，花10分钟选定1个话题，写一句核心角度
3. [Agent] 标题 Agent 根据选题+角度，生成5个标题备选
4. [人] 选定1个标题

**输出物**：一份确定的「今日选题卡」（选题+角度+标题），作为下游所有环节的输入。

**回扣方法论**：这段流程就是第3章的流程图三部曲——动作已拆解、I/O已标注、接口已对齐。提醒读者注意「选题卡」就是上下游的接力棒（呼应第5章 5.3.3 接力棒规则）。

#### 7.3.2 中午（10:00-12:00）：脚本撰写与素材准备流程

**工作流描述**：
1. [Agent] 脚本 Agent 根据「今日选题卡」生成脚本初稿
2. [人] 花20分钟修改脚本，调整口语化表达，加入个人风格的金句
3. [Agent] 根据定稿脚本，素材 Agent 推荐匹配画面 / 图像 Agent 生成所需素材
4. [人] 审核素材，标注需要自己实拍的部分
5. [人] 实拍（如果需要）
6. [Agent] TTS 生成配音（如果不是本人出镜口播）

**输出物**：定稿脚本 + 全部素材 + 配音音频 + 分镜表

#### 7.3.3 下午（14:00-16:00）：剪辑执行、发布与复盘流程

**工作流描述**：
1. [人] 按分镜表剪辑（这一步暂时人工完成，未来AI剪辑工具成熟后可部分替代）
2. [Agent] 封面 Agent 根据视频内容生成3版封面
3. [人] 选定封面
4. [人] 导出视频
5. [Agent] 生成发布文案（含话题标签）
6. [人/自动化] 发布到平台
7. [自动化脚本] 24小时后自动记录数据到表格

**输出物**：已发布的视频 + 数据记录

**回扣方法论**：整个日流程的结构就是第4章「五步法」的产物：目标明确（日更一条视频）、步骤原子化、I/O格式锁死、人机分工清晰、可复用。提醒读者：这不是一天的计划，这是一套「模具」——每天只需要往里面填不同的选题。

**展示一张「日更工作流全景图」**（时间轴 + 步骤 + 执行者 + I/O）

**【心法】**：日更的秘密不是"每天都很努力"，而是"每天都走同一条路"。当路径固定了，你的精力就可以全部投入到唯一需要创造力的环节——选题和角度。

### 7.4 案例：用流程支撑一个 IP 的长期稳定更新（约 2500 字）

**核心论点**：流程化不是扼杀创造力，而是保护创造力。只有当重复性工作被流程接管后，创作者才有精力去思考真正重要的事——内容方向和人设。

**叙事线**：回到小陈。展示他从"全靠灵感"到"流程驱动"的完整演变。

#### 7.4.1 初始状态：全靠灵感，节奏不稳定

用具体数据呈现小陈「改造前」的状态：
- 每周发布频率：2-5条（极不稳定）
- 单条视频制作时间：4-8小时
- 断更原因 Top 3：①选题想不出来 ②脚本写不下去 ③剪辑太累不想动
- 粉丝增长：缓慢且波动大
- 心理状态：焦虑、内耗、想放弃

**关键洞察**：小陈的问题不是「不够努力」，而是他把每一条视频都当成一个「从零开始的创作项目」。他没有任何可复用的东西——每次选题重新想，每次脚本重新写，每次剪辑重新摸索节奏。

#### 7.4.2 加入关键流程节点后的明显变化

小陈按照本章的方法，做了三件事：

**第一件：固化十步流程（用第3章的方法）**
- 画了一张流程图，贴在电脑旁边
- 效果：再也不会"不知道下一步干嘛"

**第二件：配置三个核心 Agent（用第5章的方法）**
- 选题 Agent：每天早上自动推送选题清单
- 脚本 Agent：输入选题卡，输出初稿
- 封面 Agent：输入视频截图，输出3版封面
- 效果：每天节省 2-3 小时

**第三件：建立试跑和迭代机制（用第6章的方法）**
- 第一周：MVP 试跑，盯着每一步的输出看
- 第二周：记录 Bug 日志（比如脚本 Agent 总写太长、选题 Agent 推荐的话题太冷门）
- 第三周：打补丁，优化 Prompt
- 第四周：流程稳定，开始放手

**改造后的数据对比**（表格呈现）：

| 指标 | 改造前 | 改造后 |
| :--- | :--- | :--- |
| 每周发布频率 | 2-5条（不稳定） | 7条（稳定日更） |
| 单条视频制作时间 | 4-8小时 | 1-2小时 |
| 选题耗时 | 1-2小时 | 15分钟（Agent推荐+人筛选） |
| 脚本耗时 | 1-2小时 | 30分钟（Agent初稿+人修改） |
| 断更次数（月） | 3-5次 | 0次 |

#### 7.4.3 形成可交付给团队执行的内容流水线

小陈粉丝增长到10万后，决定招一名助理。
- 如果没有流程：他需要花两周手把手教，助理还是学不会他的"感觉"
- 有了流程：他把「资产三件套」（流程图+Agent配置+Prompt模板）打包给助理，助理第一天就能按流程产出80%质量的内容

**回扣方法论**：这就是第4章「第五步：将工作流沉淀为可复用资产」的终极验证——你的工作流是否具备「可交接性」。

**关键金句/心法**：
> 「很多人以为做内容是"艺术"，不能标准化。但真相是：95%的爆款视频都遵循同样的结构。真正的艺术在于那5%的独特角度和个人风格。流程化不是把你变成流水线工人，而是帮你把95%的体力活甩掉，让你100%的脑力都花在那5%上。」

---

### 本章结语（约 300 字）

- 回顾全章：从"一团混沌"到"十步流水线"，从"全靠自己"到"人机协作"，从"一次性作品"到"可复用资产"
- 强调：短视频只是一个缩影。任何内容生产类工作（写文章、做播客、写报告），都可以用同样的思路去拆解
- 过渡到下一章：「如果你不做短视频，没关系。下一章我们会进入另一个高频场景——电商。你会看到同样的方法论，在一个完全不同的行业里，依然能产出惊人的效率提升。」

## 七、写作注意事项

1. **写法切换**：本章不再用前6章的「❌错误反应→💥后果→✅正确逻辑」公式。改用「小陈的故事线 + 流程拆解 + 方法论回扣」三线交织。偶尔出现的错误示范可以通过小陈的"踩坑经历"自然带出，而不是用固定格式框住。

2. **工具提及原则**：提到具体工具（如 Coze、剪映、Midjourney 等）时，只说"用什么类型的工具做什么事"，不做步骤级操作教学。避免变成工具说明书。

3. **JD 示范篇幅**：7.2 中展示的 Agent JD 只挑1个最重要的（选题 Agent 或脚本 Agent）做完整展示，其余的给出关键字段即可，避免重复感。

4. **数据真实感**：7.4 中小陈的数据对比需要符合短视频行业的合理范围，不能太夸张（比如不说"粉丝从3万涨到100万"），保持可信度。

5. **可迁移性暗示**：在每一节的末尾或心法中，偶尔提一句"这个思路不仅适用于短视频"，为非短视频读者提供代入感，但不要太刻意。
