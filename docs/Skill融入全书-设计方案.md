# Skill 融入全书：设计方案（最简版）

本书对扣子 2.0 的「Skill（技能）」只做**概念提及**：前面提一嘴、后面提一嘴，不让读者混淆，也不进入第 7～11 章实操。目的是让读者知道有这个趋势和概念，书内主线仍为「创建智能体」。

---

## 一、已实施位置

### 1. 前面：第 1 章 1.1

在「技能（Skills）」那一句后面，加一句说明，把平台上的「Skill」顺带点出，并明确本书重点在「创建智能体」：

> 在扣子等平台，这类能力正以「Skill（技能）」的形式被做成可复用模块，可被多个智能体共用或单独调用；本书主要教你创建智能体，Skill 作为延伸概念你可在熟悉智能体后自行了解。

**作用**：读者一开始就知道有「Skill」这个词和大致含义，不会和后面「创建智能体」搞混；同时声明本书不展开 Skill，避免期待错位。

### 2. 后面：全书结语（第 13 章末）

在「从手工作坊到智能工厂……」和「感谢你一路读到这里」之间，加一小段「最后顺便提一句」：

> 最后顺便提一句：眼下不少平台（如扣子 2.0）正在推「Skill（技能）」——把某一类能力封装成可复用模块，供多个智能体配置或单独调用。你可以理解为：智能体像你招来的**一个员工**（有名字、有岗位，你跟他对话、给他派活）；Skill 像公司里**谁都能领用的一套标准工具或操作手册**（比如「查订单」「生成报表」），同一套可以配给不同的员工，也可以单独拿出来用。二者互补，本书专注教你搭智能体与工作流，Skill 作为平台新方向，留待你在用熟智能体之后再去探索即可。

**作用**：收束时点明概念和趋势，再次区分「智能体 vs Skill」，并明确「书里不教 Skill 实操、读者日后可自行探索」。

## 三、表述约定（若日后需在其他处提及）

- **智能体 / Agent**：像你招来的一个员工，有名字、有岗位，你跟他对话、派活；本书教的「创建智能体」= 搭一个这样的员工/助手。
- **Skill（技能）**：像公司里谁都能领用的标准工具或操作手册（如查订单、生成报表），同一套可配给不同员工，也可单独用；平台上的可复用能力模块。
- 比喻区分：**智能体 = 员工，Skill = 工具/手册；同一套工具可以配给不同员工，本书只教搭智能体。**
