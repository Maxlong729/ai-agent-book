## 一、书籍定位与目标（来自写书Prompt）

- **书名**：《效率倍增的 AI Agent 工作流：从零搭建一套你的智能体工作方式》
- **一句话定位**：写给「已经在用 AI，但始终用不深」的人，教他们把 AI 从「偶尔帮忙的工具」升级为【长期稳定产出、可复制、可迁移的个人工作系统】。
- **目标读者**：不是 AI 小白，而是——内容创作者/电商/教育/职场/创业者、AI 工具重度但「用得碎、用得浅」的人；共同状态是工具一堆流程乱、会用 Prompt 但每次从头想、AI 能帮忙但无法持续替自己干活。
- **核心要解决的问题**：不是「会不会用 AI」，而是「为什么已经很努力用 AI，却始终没形成体系」。

**读完后的变化**（每一章至少要推动其中一条）：
1. **认知**：不再把 AI 当工具，开始用「工作流/系统/Agent」思考。
2. **做事方式**：做任何事前先想「这件事能不能变成可复用流程？」；把重复性脑力劳动交给 AI 体系。
3. **身份感**：从「到处救火的执行者」变成「有一套自己工作方式的人」，效率与掌控感提升。

## 二、写作原则与红线（来自写书Prompt）

**角色与优先级**：同时扮演畅销书作者、出版社责编、系统型内容策划；冲突时优先级：**读者真实收获 > 是否好读 > 是否写得完整**。宁可少写、删写，也不许「写了但没用」。

**市场与出版**：本书**不是** AI 教程/工具大全/技术原理/培训教材；**而是**「面向真实工作场景的认知+方法+结构书」。写作标准：可以不严谨但不能抽象，可以不全面但必须打透；每一章可单独阅读；前 3 页要让读者觉得「和之前看的 AI 书不一样」。

**语言与表达**：
- 第一、二人称为主；口语化但不水；有明确判断，不做中立总结；偶尔犀利不说教。
- 一个自然段 = 一个明确结论；出现抽象词必须立刻落到具体工作场景；不解释「AI 是什么」，直接讨论「你该怎么用、为什么你现在用不好」。
- **绝对禁止**：正确但没用的总结、「未来趋势」空话、工具堆砌名词轰炸、把流程写成理想状态忽略现实复杂度。

**结构**：每一章只回答一个问题；每一章必须回答「这一章帮读者在哪一步少走弯路？」。不追求覆盖所有 AI 能力，只聚焦能真正改变工作方式的关键节点；宁可删掉 30% 也不保留「看起来很完整」的内容。

**协作**：写正文前先给章节结构；发现「对但无聊」必须提醒；发现可删内容不用请示直接删；某一章不值得写可以直接说。职责是「写对」不是「多写」。

**启动确认**（每章开始前）：用不超过 200 字确认：① 本书核心价值是什么？② 这一章解决读者哪一个具体卡点？③ 读完这一章读者会在哪件事上明显轻松？确认后再写正文。

## 三、实操章节写作金标准（基于第 5 章风格）

凡涉及「具体方法」或「案例」的章节，**必须**严格遵循以下 5 大特征：

1.  **强烈的对象感**：全程用「你」对话。多用「想象一下...」「你肯定遇到过...」做场景钩子，唤起痛点记忆。
2.  **红黑榜对比（❌ vs ✅）**：拒绝平铺直叙。先展示「错误的本能反应」及其惨痛后果，再给出「正确的工程逻辑」，通过反差制造认知冲击。
3.  **填空式交付**：不教原理，只给工具。提供**「通用模板」+「实战案例」**，设计好字段让读者做填空题。
4.  **犀利笃定的语气**：多用短句、断言。使用「死规矩、红线、保命符」等强力词汇，建立专家安全感。
5.  **具象化比喻**：将技术概念物理化。如：Agent=实习生、Prompt=岗位说明书、流程传递=接力棒。

**核心心法**：不是在写书，是在做「产品说明书」。

## 四、样章要点（第 5 章：给 Agent 写一份清楚的「岗位说明书」）

**5.1 用四个要素定义一个 Agent 的岗位**
- **工作目的**：给谁看？用来干嘛？什么叫合格？不写形容词，写「动作」和「对象」。
- **输入来源**：允许用哪些材料？红线：不得自行补充事实，信息缺失先列「问题清单」。
- **输出成果**：成品长什么样？最好「定字段」、当填空题，规定板块与要素。
- **限制条件**：哪些事绝对不能替你做主？如不承诺、不归责、不推测；兜底机制。

**5.2 为不同类型 Agent 设计不同岗位**
- **信息类 Agent**：只做搜集与整理，交付「可核对的资料包」；死规矩：找不到也不许编。
- **分析类 Agent**：基于你给的事实与逻辑推导结论；死规矩：结论有据可依，不凭空「觉得」。
- **写作类 Agent**：把已确定的要点填进固定结构；死规矩：结构第一，文采第二。

**5.3 何时拆成多个 Agent**
- 关键词检测法：工作目的里出现「然后」「并且」「顺便」「同时」→ 建议拆分。
- 将「搜集」与「判断」拆分，避免认知混流、自圆其说。
- 将「写作」与「改写」拆分，避免发散与克制冲突。
- 拆分后设计「接力棒」：上游输出规定总结清单，下游只读该清单，缺信息就报错不编。

**5.4 可直接复用的岗位说明书**
- 万能填空表：Role / Profile / Goals / Constraints / Input / Output 等；强调可复制、可版本管理。

**样章风格**：每节只解决一件事；用❌错误写法 / ✅正确逻辑 + 通用模板 + 周报 Agent 示范；有表格/模板框；口语化、有判断、落到具体场景。
